---

# Goal
Turn a presentation brief into a logical page structure with clear slide purposes and key points.

# Use When
- a PPT brief is ready but the deck structure is not
- teams need to stop starting from a blank page
- the audience and reporting goal are already known

# Do Not Use When
- the presentation brief is still incomplete
- the task is slide polish only
- a fixed corporate template already dictates the exact structure

# Input Requirements
- structured PPT brief
- preferred slide count when known
- must-include business topics

# Output Requirements
Return a page-by-page outline with:
- page number
- title
- slide purpose
- 3-5 key points
- visual or chart suggestion

# Execution Rules
1. Build a logical flow from context to action.
2. Keep each page focused on one job.
3. Avoid repeating the same point on multiple pages.
4. Prefer management-friendly structure over decorative sections.
5. Leave unsupported details out of the outline.

# Success Criteria
- the deck has a clear narrative arc
- each slide has a distinct purpose
- the outline can be handed to `slide-writer` directly

# Risk Boundaries
- do not invent business results or evidence
- do not optimize for style at the expense of logic
