# 文案渠道必保留字段表

## 目标
把不同渠道最容易丢失的关键字段明确下来，避免主结果生成时因为压缩篇幅而漏掉价格、日期、联系方式或动作提示。

## 使用原则
1. 先判断当前属于“对外推广”还是“内部通知”。
2. 生成前先看该渠道的必保留字段。
3. 如果长度不够，优先压缩修饰语和次要描述，不要先删关键字段。
4. 如果关键字段无法同时保留，必须在审校结果里显式标出，让人工判断。

## 对外推广场景

| 渠道 | 必保留字段 | 可弱化字段 | 审校重点 |
| ---- | ---------- | ---------- | -------- |
| 海报标题 `poster_title` | 活动主题或核心利益点 | 次要卖点、联系方式 | 是否过长、是否一眼能看懂 |
| 朋友圈文案 `moments_post` | 时间、截止时间、价格、核心卖点、联系方式或报名入口 | 次要氛围描述 | 是否漏掉报名信息，是否像硬广 |
| 社群通知 `group_notice` | 时间、截止时间、价格、对象、核心卖点、联系方式或报名入口 | 额外情绪化描述 | 信息是否完整、是否适合群内转发 |
| 短信 `sms` | 活动名或活动简称、时间或截止时间、一个关键行动、联系方式或报名入口 | 次要卖点、长句解释 | 是否超长、是否还能看懂 |

## 内部通知场景

| 渠道 | 必保留字段 | 可弱化字段 | 审校重点 |
| ---- | ---------- | ---------- | -------- |
| 通知标题 `notice_title` | 通知主题、假期或事项类型 | 修饰语 | 是否清楚、是否正式 |
| 企业微信通知 `wechat_group_notice` | 时间安排、返岗或执行时间、动作要求、联系人 | 礼貌性寒暄 | 是否清楚、是否遗漏动作提示 |
| 邮件通知 `email_notice` | 时间安排、动作要求、例外或紧急联系人 | 祝福语、背景说明 | 是否完整、是否适合作为正式留档 |
| 短信 `sms` | 时间安排、关键动作、联系人 | 次要解释 | 是否超长、是否还能执行 |

## 缩写优先级
当文本过长时，压缩顺序建议如下：

1. 删掉重复修饰语
2. 合并重复卖点
3. 把次要说明移动到更长渠道
4. 保留关键事实和动作提示

## 绝对不要先删的内容
- 价格
- 日期
- 截止时间
- 联系方式
- 报名入口
- 紧急联系人
- 明确动作要求
