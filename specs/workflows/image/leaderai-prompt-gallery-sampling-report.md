# Leader Prompt Gallery 抽样分析报告

## 目标
对 [Leader Prompt Gallery](https://www.leaderai.top/mid-api/lab/image_prompt/index.html) 进行分类抽样分析，提炼其中高频 prompt 结构、常见任务类型和适合并入当前仓库图片 workflow 的模式。

如果需要覆盖全部 `16` 个分类，再看：
- [leaderai-prompt-gallery-all-category-sampling-report.md](D:/work/myclaw/codex-personal-agent-starter/specs/workflows/image/leaderai-prompt-gallery-all-category-sampling-report.md)

## 数据范围与方法

### 数据来源
- 页面入口：`https://www.leaderai.top/mid-api/lab/image_prompt/index.html`
- 页面实际数据源：`catalog.json`
- 总条目数：`6644`

### 本次分析不是做什么
- 不是逐条人工阅读 `6644` 条模板
- 不是全量内容精读
- 不是做最终质量排名

### 本次分析做了什么
1. 确认全站分类分布
2. 选择与当前仓库最相关的主要类别做抽样
3. 对全量数据做关键词模式统计
4. 结合代表性样本判断高频 prompt 写法

## 全量模式统计

以 `6644` 条记录为范围，用关键词规则做粗粒度统计，得到：

- `structured_meta`：2502 条，约 `37.7%`
- `style_control`：1906 条，约 `28.7%`
- `reference_based`：763 条，约 `11.5%`
- `negative_control`：634 条，约 `9.5%`
- `composition_control`：395 条，约 `5.9%`
- `transformation_task`：297 条，约 `4.5%`

说明：
- `structured_meta`：指模板中明显出现 `json`、`meta`、`role`、字段化描述等结构
- `style_control`：明显包含风格、电影感、写实、3D、插画、极简等风格控制词
- `reference_based`：明显依赖上传图、参考图、图1图2、保持不变、recreate 等
- `negative_control`：明确写出不要什么、避免什么
- `composition_control`：明确提到比例、留白、构图、标题区
- `transformation_task`：偏编辑、迁移、提取、修复、替换类任务

## 分类抽样观察

### 1. 人物肖像与写实摄影
样本量：`2771`

模式统计：
- `structured_meta`：1637，约 `59.1%`
- `style_control`：801，约 `28.9%`
- `reference_based`：359，约 `13.0%`

代表样本：
- `高还原度黑白暗调人像摄影`
- `喧闹人群中静止清晰的电影感肖像`
- `动静对比的高级时尚杂志人像`

结论：
- 这一类最强的是“人物主体 + 光线情绪 + 质感目标”
- 对“电影感、杂志感、真实写真”的描述很高频
- 常见写法不是单个风格词，而是：
  - 主体身份
  - 姿态
  - 镜头氛围
  - 光线
  - 表情
  - 画面控制

对仓库的启发：
- 写真人像 prompt 必须强调：
  - 人物主体
  - 表情与姿态
  - 光线
  - 皮肤和质感边界
- 不能只写“高级感、电影感、真实感”

### 2. 创意脑洞特效合成
样本量：`936`

模式统计：
- `structured_meta`：301，约 `32.2%`
- `style_control`：290，约 `31.0%`
- `reference_based`：89，约 `9.5%`

代表样本：
- `双重曝光：融合自然元素的动物剪影`
- `肌肉与森林光影交融的视觉艺术`
- `创意团队协作场景：数字与符号形会议桌上的高效工作构图`

结论：
- 这类模板非常依赖“核心视觉创意句”
- 经常把两个不相关元素做隐喻叠加
- 风险最大的是画面容易花、容易噱头化

对仓库的启发：
- 可以把它吸收到“创意方向探索”，但不能拿来当默认商业主方向
- 需要单独加强 negative prompt，对“杂乱、过度特效、过度概念化”做压制

### 3. 创意广告品牌设计
样本量：`306`

模式统计：
- `style_control`：117，约 `38.2%`
- `structured_meta`：112，约 `36.6%`
- `reference_based`：34，约 `11.1%`

代表样本：
- `8位像素复古游戏图标`
- `包豪斯风格重塑品牌标志设计`
- `可口可乐瓶置于爆炸中心的电影级商业海报`

结论：
- 这一类是“高完成度广告 prompt”的主产地
- 结构上经常出现：
  - 产品或品牌主体
  - 风格化表现方式
  - 场景装置
  - 高冲击广告感
- 同时也最容易出现“概念很强但品牌信息弱”的问题

对仓库的启发：
- 适合强化当前 `海报主视觉` 和 `品牌 KV` 类方向
- 需要保持你仓库已有的 brief 约束，不然容易只剩概念画面

### 4. 艺术插画创意风格
样本量：`244`

模式统计：
- `style_control`：96，约 `39.3%`
- `structured_meta`：54，约 `22.1%`
- `reference_based`：46，约 `18.9%`

代表样本：
- `复古风黑白人像摄影艺术效果`
- `极简扁平插画风格设计模板`
- `抽象乱线人像艺术创作`

结论：
- 这一类更偏“风格变体库”
- 很适合做方向 B / 方向 C 这种备用探索
- 但不适合直接替代商业主方向

对仓库的启发：
- 可以把这类模式吸收进 `image-prompt-generator` 的“创意方向”分支
- 但必须保留 brand review 过滤

### 5. 卡通漫画电影角色
样本量：`185`

模式统计：
- `style_control`：67，约 `36.2%`
- `reference_based`：35，约 `18.9%`
- `structured_meta`：13，约 `7.0%`

代表样本：
- `萌系3D动物角色设计模板`
- `萌系Q版卡通人物，薄荷绿运动风，手持珍珠奶茶，3D可爱形象`
- `海滩排球少女跃起扣杀瞬间`

结论：
- 这类模板更依赖“角色设定词”和“外观描述”
- 比起广告类 prompt，这类更强调：
  - 发型
  - 服装
  - 表情
  - 姿态
  - 角色气质
- 但 negative prompt 普遍不够强

对仓库的启发：
- 当前仓库在“角色图”上最应该补的，不是更多风格词，而是：
  - 角色描述骨架
  - 风险压制词
  - 二轮修改模板

### 6. 图像编辑与修复增强
样本量：`100`

模式统计：
- `transformation_task`：34，约 `34.0%`
- `reference_based`：23，约 `23.0%`

代表样本：
- `老照片AI修复还原为专业人像摄影效果`
- `老照片修复为专业级彩色人像`
- `将动漫风格图像转为真实照片效果`

结论：
- 这类模板更像“编辑指令库”
- 结构常见为：
  - 保留什么不变
  - 改什么
  - 目标输出风格
- 非常适合当前仓库的 `image-result-reviewer` 和 `ChatGPT Pro 二轮改图`

## 高价值模式总结

### 模式 1：字段化 prompt 很常见
抽样和全量统计都说明，大量模板已经不再是自然语言一整段，而是：
- `json`
- `meta`
- `role`
- `scene`
- `style`
- `subject`

这说明：
- 高质量 prompt 已经在向“半结构化描述”迁移
- 你仓库保留 `visual brief -> direction prompt` 这条链路是对的

### 模式 2：参考图 / 编辑类任务占比不低
全量里约 `11.5%` 明确是参考图驱动，编辑/替换/迁移类任务也有稳定比例。

这说明：
- 图片 workflow 不能只做“从零生成”
- 必须为“参考图编辑”和“保持主体不变”留结构位

### 模式 3：negative prompt 远比站点表面看起来更重要
虽然显式 negative prompt 的比例不算特别高，但高质量样本里几乎都隐含了“风险压制”：
- 不廉价
- 不卡通
- 不拥挤
- 不塑料
- 不影楼

这说明：
- 当前仓库继续做 negative prompt 词库是值得的
- 尤其在人像、角色图、广告图这三类任务里

### 模式 4：比例、留白、标题区控制经常被低估
全量里显式构图 / 比例控制约 `5.9%`，看起来不高，但在商业可用图里非常关键。

这说明：
- 真正能进生产的图，不只是“画得像”
- 还要：
  - 有留白
  - 可上标题
  - 可适配 9:16 / 16:9 / 1:1

## 对当前仓库的直接建议

### 建议 1：保留现有 workflow 主结构
不要被外部 prompt 库带偏。

当前仓库仍应坚持：
1. 最小补问
2. visual brief
3. 方向与 prompt
4. brand review
5. 人工确认

### 建议 2：把 prompt 模式库定位成“执行层参考”
也就是现在的：
- [image-prompt-pattern-library.md](D:/work/myclaw/codex-personal-agent-starter/specs/workflows/image/image-prompt-pattern-library.md)

它应该负责：
- prompt 写法模式
- negative prompt 片段
- ChatGPT Pro 手动出图拼装方式

而不是替代 brief。

### 建议 3：下一轮重点补两类资产
优先补：
1. `角色图 prompt 模板`
2. `写真类 negative prompt 词库`

因为这两个是当前仓库最常用、也最容易跑偏的。

### 建议 4：把“参考图编辑”单独设成图片 workflow 子场景
因为这个库里参考图驱动的任务比例不低，适合后续单独加一个：
- `reference_edit`

## 结论
Leader Prompt Gallery 对当前仓库最有价值的，不是“提供 6644 条现成 prompt”，而是证明了几件事：

1. 高质量 prompt 越来越结构化
2. 参考图编辑是主流任务，不是边缘需求
3. 风格词不够，必须带约束和风险压制
4. 商业可用图一定要提前考虑比例、构图和留白

所以最优策略不是全盘照搬，而是：
- 保留你仓库自己的 workflow 主结构
- 吸收这个库里的 prompt 模式和风险写法
- 继续把它沉淀成你自己的 prompt 资产层

## 局限说明
- 本报告不是 6644 条全量人工精读
- 分类统计采用关键词规则，属于模式识别，不是精确语义标注
- 代表样本只用于辅助判断，不代表整个分类的全部质量水平
