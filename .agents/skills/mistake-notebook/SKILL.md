# 错误笔记

## 位置

错误笔记有两个位置：

- 项目级错误笔记：项目根目录下的 `MISTAKE.md`
- 全局错误笔记：用户主目录下的 `~/MISTAKE.md`

项目级错误优先记录与本仓库直接相关的问题；全局错误用于记录通用、跨项目的经验。

## 何时使用

### 1. 遇到严重问题

- 先检查 `MISTAKE.md` 和 `~/MISTAKE.md` 是否已有类似问题
- 如果已有记录：
  - 先尝试按既有方案处理
  - 如果无效，再分析为什么旧方案失效，并在解决后更新记录
- 如果没有记录：
  - 自行解决问题
  - 解决后进入“记录错误方案”

### 2. 修复了严重错误

满足以下条件时应记录：

- 问题已经解决
- 原因和解决方案足够清楚
- 对以后有复用价值

### 3. 用户明确要求“不要再犯”

- 如果问题还没解决，先解决
- 解决后把问题和方案写入错误笔记

## 如何记录

- 先检查项目级和全局错误笔记中是否已存在相同问题
- 如果已有但内容过时，更新原记录
- 如果没有：
  - 项目相关问题写入 `MISTAKE.md`
  - 通用问题写入 `~/MISTAKE.md`

## 记录格式

```markdown
# 错误笔记

## 错误 - 简短标题

- 创建日期：YYYY-MM-DD
- 更新日期：YYYY-MM-DD
- 项目：项目路径
- 分支：分支名
- 提交：提交 SHA

### 问题

- 何时发生
- 具体现象
- 错误表现

### 发现

- 排查过程中得到的关键认识

### 解决方案

- 一句话总结解决方法
- 再列出简短步骤
```

## 边界

- 如果错误笔记不存在，就创建
- 如果存在，就追加或更新
- 如果因为权限无法写入，就明确告诉用户并给出待写入内容
