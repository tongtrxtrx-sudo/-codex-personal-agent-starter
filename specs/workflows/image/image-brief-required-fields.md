# 图片 Brief 必保留字段表

## 目标
明确图片 workflow 在不同阶段不能丢的关键信息，避免后续 prompt 只剩风格词、没有业务事实。

## Step 1：视觉 Brief 抽取时必须保留

| 字段 | 必要性 | 说明 |
| ---- | ------ | ---- |
| `subject` | 必填 | 画面主体是什么 |
| `audience` | 必填 | 图给谁看 |
| `scene` | 必填 | 发生在什么场景 |
| `style_keywords` | 必填 | 风格方向，不少于 2 个 |
| `brand_tone` | 必填 | 品牌感受，例如亲切、可信、专业 |
| `must_show_elements` | 建议必填 | 画面必须出现什么 |
| `text_overlay` | 建议必填 | 需要上版的文案或事实 |
| `negative_constraints` | 必填 | 明确不希望出现什么 |

## Step 2：方向与 Prompt 生成时必须保留

每个方向都必须保留：
- 方向名
- 视觉描述
- Prompt
- Negative Prompt
- 主标题建议
- 风险提醒

## Step 3：品牌审校时必须检查
- 受众是否匹配
- 品牌调性是否匹配
- 是否存在廉价感或过度卡通
- 标题是否过长或过促销
- 必保留事实是否仍保留

## 绝对不要先删的内容
- 受众
- 场景
- 品牌调性
- 必保留事实
- 负向限制
