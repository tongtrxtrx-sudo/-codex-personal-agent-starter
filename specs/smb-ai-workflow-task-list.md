# 中小企业 AI Workflow 最佳实践任务清单

> 来源规格：[smb-ai-workflow-best-practice-plan.md](D:/work/myclaw/codex-personal-agent-starter/specs/smb-ai-workflow-best-practice-plan.md)
> 使用方式：按阶段推进，完成一项就勾掉；未完成项不要提前跨阶段扩张。

## 当前目标

- [ ] 固定“中小企业 AI 工作流培训与落地顾问”的对外定位
- [ ] 只保留 2 类产品线
- [ ] 只推进 4 条 Workflow
- [ ] 固化 12 个核心 Skills
- [ ] 完成一轮试讲可交付版本

## 第一阶段：定位与边界冻结

### A. 对外定位
- [ ] 写出 1 句统一定位文案
- [ ] 写出 1 段老板版自我介绍
- [ ] 写出 1 段员工培训版自我介绍
- [ ] 明确“不主打万能 Agent、不主打复杂技术”的边界表述

### B. 产品线收束
- [ ] 固定产品 1：`中小企业 AI 工作流入门培训`
- [ ] 固定产品 2：`行政办公与内容生产 AI 实操培训`
- [ ] 为每个产品写清目标对象
- [ ] 为每个产品写清核心收益
- [ ] 为每个产品写清不覆盖的范围

### C. 首批场景冻结
- [ ] 固定 Workflow 1：文案
- [ ] 固定 Workflow 2：图片
- [ ] 固定 Workflow 3：视频
- [ ] 固定 Workflow 4：PPT
- [ ] 明确第一阶段不新增第 5 条 Workflow

### 第一阶段完成标准
- [ ] 所有对外表达都统一为“先 Skill，再 Workflow，最后 Agent”
- [ ] 两类产品的边界清楚，不再混讲
- [ ] 4 条 Workflow 名单冻结

## 第二阶段：Skill 标准化

### A. 文案 Workflow
- [ ] 定义 `brief_extractor`
- [ ] 定义 `copy_generator`
- [ ] 定义 `copy_reviewer`

### B. 图片 Workflow
- [ ] 定义 `visual_brief_extractor`
- [ ] 定义 `image_prompt_generator`
- [ ] 定义 `brand_checker`

### C. 视频 Workflow
- [ ] 定义 `video_angle_generator`
- [ ] 定义 `script_writer`
- [ ] 定义 `storyboard_reviewer`

### D. PPT Workflow
- [ ] 定义 `ppt_brief_extractor`
- [ ] 定义 `deck_outliner`
- [ ] 定义 `slide_writer`

### E. 每个 Skill 都要补齐
- [ ] 名称
- [ ] 适用场景
- [ ] 不适用场景
- [ ] 输入格式
- [ ] 输出格式
- [ ] 执行规则
- [ ] 成功标准
- [ ] 风险边界

### 第二阶段完成标准
- [ ] 12 个 Skills 都有统一模板
- [ ] 没有职责混杂的 Skill
- [ ] 所有 Skill 都有清晰边界

## 第三阶段：Workflow 结构固化

### A. 统一结构
- [ ] 文案 Workflow 改成 4 步结构
- [ ] 图片 Workflow 改成 4 步结构
- [ ] 视频 Workflow 改成 4 步结构
- [ ] PPT Workflow 改成 4 步结构

### B. 共同 4 步
- [ ] Step 1：Brief 抽取
- [ ] Step 2：主结果生成
- [ ] Step 3：审校与优化
- [ ] Step 4：人工确认

### C. 风险与人工确认
- [ ] 明确价格类内容必须人工确认
- [ ] 明确日期类内容必须人工确认
- [ ] 明确联系方式必须人工确认
- [ ] 明确品牌口径必须人工确认
- [ ] 明确预算或决策建议必须人工确认
- [ ] 明确对外发布内容必须人工确认

### 第三阶段完成标准
- [ ] 4 条 Workflow 全部采用统一结构
- [ ] 每条 Workflow 都有清晰人工确认点
- [ ] 不存在“生成即最终稿”的节点

## 第四阶段：测试与评估

### A. 连续测试
- [ ] 文案 Workflow 连跑 5 次
- [ ] 图片 Workflow 连跑 5 次
- [ ] 视频 Workflow 连跑 5 次
- [ ] PPT Workflow 连跑 5 次

### B. 每轮都检查
- [ ] 是否漏掉关键字段
- [ ] 是否丢失价格、日期、联系方式
- [ ] 是否风格不稳定
- [ ] 是否输出格式漂移
- [ ] 是否出现夸张表达
- [ ] 是否出现高频返工点

### C. 问题回灌
- [ ] 把高频问题补到 Reviewer 或 Checker
- [ ] 修正不稳定的输入字段
- [ ] 修正不稳定的输出结构
- [ ] 形成一版问题清单

### 第四阶段完成标准
- [ ] 4 条 Workflow 都完成至少 5 次测试
- [ ] 问题清单可复查
- [ ] 已做一轮基于问题的修正

## 第五阶段：演示与培训交付

### A. Demo 打磨
- [ ] 用统一案例串起 4 个 Demo
- [ ] 每个 Demo 控制在 5-7 分钟
- [ ] 把 4 个 Demo 串成 30 分钟演示
- [ ] 每个 Demo 都能讲清业务痛点、流程、结果、人工边界

### B. 标准交付件
- [ ] 培训 PPT
- [ ] 讲师逐字稿
- [ ] 4 个 Demo 示例
- [ ] 常用 Prompt 模板
- [ ] 企业 AI Workflow 落地建议
- [ ] 课后练习题
- [ ] 课后模板包

### C. 试讲验证
- [ ] 做 1 次老板版试讲
- [ ] 做 1 次员工版试讲
- [ ] 收集听众是否能复述“Skill -> Workflow -> Agent”
- [ ] 收集听众是否能说出至少 1 个自己岗位可用场景
- [ ] 根据反馈修正文案和讲法

### 第五阶段完成标准
- [ ] 形成完整标准交付包
- [ ] 能完成一轮对外试讲
- [ ] 讲法足够通俗，不偏技术炫技

## 第六阶段：Agent 进入条件

> 这一阶段不是现在就做，而是满足条件后再进入。

- [ ] 4 条 Workflow 已稳定
- [ ] 试讲反馈显示 Workflow 方法已被理解
- [ ] 已有明确“多 Workflow 路由”需求
- [ ] 已经具备人工确认和风险边界
- [ ] 再决定是否加入总控 Agent 演示

## 今天就能开始的 3 个动作

- [ ] 写出统一定位文案
- [ ] 写出两类产品的课程简介
- [ ] 为 4 条 Workflow 列出 12 个 Skills 的标准模板骨架

## 暂时不要做

- [ ] 不扩更多行业版课程
- [ ] 不扩更多岗位版课程
- [ ] 不做万能 Agent 方案
- [ ] 不做自动外发
- [ ] 不引入额外平台化建设

## 完成判定

- [ ] 你能用一句话解释你的定位
- [ ] 你能清楚区分两类产品
- [ ] 你能把 4 条 Workflow 从头讲到尾
- [ ] 你手上有一套可直接交付的培训资料包
- [ ] 你不是在卖“会用 AI”，而是在卖“可落地的 AI Workflow 方法”
