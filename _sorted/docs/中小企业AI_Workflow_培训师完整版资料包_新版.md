# 三、企业最佳实践总纲

## 1. 一句话原则

**能固定就 Workflow，需判断才 Agent；能复用就做成 Skill；先评估再扩张；高风险必须审批和人工兜底。**

---

## 2. 三层方法论

### Skill
最小可复用能力单元。  
例如：
- brief 抽取
- 文案生成
- 文案审校
- PPT 目录生成
- 视频脚本生成

### Workflow
多个 Skills 按固定顺序组成的处理链。  
例如：
- 活动需求 → 文案 → 审校
- 活动需求 → 视觉方向 → prompt → 品牌检查
- 活动资料 → PPT 目录 → 页面内容 → 演讲备注

### Agent
任务调度者。  
它负责：
- 判断调用哪条 workflow
- 管理多 workflow 路由
- 决定何时停下等待人工确认

---

## 3. 企业落地最佳顺序

### 第一阶段
先做高频、低风险、可标准化场景的 Skill。

### 第二阶段
把多个 Skill 串成 Workflow。

### 第三阶段
在 workflow 稳定之后，再加总控 Agent。

### 第四阶段
建立评估、审校、审批和复盘机制。

---

## 4. 为什么先 Workflow 再 Agent

因为对中小企业来说，大多数真实场景的关键问题不是“模型不够聪明”，而是：

- 需求不清
- 步骤不清
- 输出不稳定
- 没有审校
- 没有人负责最后确认

Workflow 能先解决“流程可控”问题。  
而 Agent 更适合在流程成熟之后，负责更复杂的路由和调度。

---

## 5. 共同结构

所有企业内容生产 workflow 都建议采用这 4 步：

1. Brief 抽取  
2. 主结果生成  
3. 审校与优化  
4. 人工确认

# 六、每个 Demo 统一讲法模板

为了让你的试讲更稳，建议每个 Demo 都固定按同一节奏来讲：

## 1. 先讲业务痛点
例如：
- 文案：不是不会写，而是多渠道改写很耗时间
- 图片：不是不会做图，而是方向说不清
- 视频：不是不会剪，而是策划链断掉
- PPT：不是不会排版，而是逻辑整理太耗时间

## 2. 展示原始输入
让学员看到现实里拿到的需求本来就是杂乱的。

## 3. 先做 Brief 抽取
强调：第一步不是生成，而是标准化。

## 4. 再做主结果生成
让大家看到 AI 的初稿能力。

## 5. 再做审校和优化
让大家看到 workflow 不是一次性问答。

## 6. 最后强调人工确认
让企业理解 AI 的边界。

# 十二、常见答疑话术

## 问：是不是以后只要一个 Agent 就可以？
答：
不是。中小企业第一阶段最稳的做法通常不是做一个万能 Agent，而是先把几个高频工作做成固定 workflow，再由 Agent 做路由和调度。

## 问：是不是提示词越复杂越好？
答：
不一定。真正稳定的效果，往往来自任务拆解清楚、输入输出定义清楚，而不是 prompt 写得越长越复杂。

## 问：企业最适合先从哪些场景开始？
答：
通常建议从：
- 文案
- 会议纪要
- PPT 初稿
- 视频脚本
- 报表总结
- 客服回复
这些高频、低风险、可标准化场景开始。

## 问：AI 会不会替代人？
答：
更现实的说法不是替代，而是重组工作方式。AI 更适合先接手重复、标准化、耗时间的部分，人继续负责判断、决策和对外承担责任的部分。

# 十三、讲师练习建议

## 1. 第一步：先练统一开场
不要一开始就练所有细节，先把统一开场讲顺。

## 2. 第二步：逐个练 Demo
按文案、图片、视频、PPT 的顺序逐个练。

## 3. 第三步：控制每个 Demo 在 5-7 分钟
这样节奏更适合企业试听与试讲。

## 4. 第四步：录屏回看
重点检查：
- 是否太技术化
- 是否讲清业务价值
- 是否强调人工确认
- 是否把 workflow 和单次问答的区别讲透

## 5. 第五步：做一版自己的语言风格
逐字稿是底稿，不是最终定稿。练熟后要慢慢讲成自己的语言。
