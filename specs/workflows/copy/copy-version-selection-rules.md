# 文案版本选择规则

## 目标
减少“版本很多但不知道选哪个”的问题，让文案 workflow 在生成后能更快给出主推版本和备选版本。

## 总原则
1. 默认不是“越多版本越好”，而是“版本分工清楚”。
2. 高风险场景优先选最稳版本，不优先选最花的版本。
3. 如果用户没说明偏好，优先推荐最完整、最稳、最容易直接发布的版本。

## 一、海报标题选择规则

### Version 1：直接型
- 默认主推版本
- 适合：
  - 正式招生
  - 正式推广
  - 需要快速识别主题

### Version 2：利益或情绪切入型
- 适合作为补充版本
- 适合：
  - 需要更有吸引力
  - 需要更生活化
  - 海报主画面已经能补充主题信息

### Version 3：动作导向型
- 适合：
  - 报名中
  - 限量
  - 预约提醒
  - 截止提醒

### 默认推荐顺序
- 招生推广：优先 V1，其次 V3，最后 V2
- 内部通知：优先 V1，其次 V3，最后 V2
- 服务提醒：优先 V1 或 V3，视动作强弱决定

## 二、朋友圈文案选择规则

### Version 1：完整标准版
- 默认主推版本
- 必须完整保留时间、截止、价格、入口
- 适合大多数直接发布场景

### Version 2：侧重点替代版
- 用于强调另一个卖点或语气方向
- 不能因为换侧重点而丢关键信息

### 默认推荐顺序
- 用户没说明偏好：优先 V1
- 用户希望更生活化：先看 V2
- 用户希望更稳更全：只推 V1

## 三、通知类版本选择规则

### 标题
- 默认优先最直白版本
- 时间型标题只做备选，不做默认主标题
- 动作型标题适合报名或返岗提醒

### 正文
- 企业微信通知：优先清楚和简洁
- 邮件通知：优先完整和正式
- 短信提醒：优先时间、动作、联系人

## 四、何时必须给出“推荐采用版本”

以下情况，review 阶段必须明确推荐版本：
- 有多个标题版本
- 有多个朋友圈文案版本
- 用户没有明确偏好
- 某个版本明显更稳或更适合当前场景

## 五、不推荐版本的常见原因
- 过长
- 关键信息缺失
- 过度情绪化
- 过度营销化
- 太空泛，看不出主题
- 和当前场景不匹配

## 六、推荐输出格式

在 review 结果里补一段：

```md
## 推荐采用版本
- 海报标题：版本 X
- 朋友圈文案：版本 X
- 推荐理由：
  - 最完整
  - 最稳
  - 最适合当前场景
```
