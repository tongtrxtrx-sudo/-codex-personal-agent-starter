# 文案初稿

## 标题
- 中小企业做 AI，为什么要先从 skill 开始

## 导语
很多企业一提 AI，就想直接上 Agent。可真正落地时，最容易卡住的，往往不是模型能力，而是动作本身并不稳定。

## 正文结构
### 一、为什么一开始不适合直接上 Agent
- 因为流程还没跑顺
- 因为动作还没稳定
- 因为问题往往不是“模型不够聪明”，而是“输入和步骤不清楚”

### 二、为什么先做 Skill
- Skill 是最小可复用动作
- 它先把高频、重复、可标准化的部分沉淀下来
- 它比直接做大而全的 Agent 更稳、更容易验证

### 三、什么时候再升级到 Workflow
- 当多个 Skill 已经稳定
- 当上下游输入输出开始清楚
- 当你知道哪些环节必须人工确认

### 四、什么时候才适合 Agent
- 当流程已经成熟
- 当你真的需要多条链路调度
- 当你知道 Agent 应该帮你做“路由”，而不是替你包办一切

## 结尾引导
中小企业做 AI，最稳的路径不是一开始追求万能，而是先把动作做稳，再把流程跑顺。先 skill，再 workflow，最后 agent。
