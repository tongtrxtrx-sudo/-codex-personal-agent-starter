# 文案渠道长度约束表

## 目标
为不同渠道给出可操作的长度建议，减少标题过长、短信过密、社媒文案失衡的问题。

## 使用原则
1. 长度约束优先用于生成前自检和审校。
2. 这是工作流约束，不是法律限制，但默认应遵守。
3. 如果必须超出建议长度，必须在审校结果中说明原因。

## 对外推广场景

| 渠道 | 建议长度 | 说明 |
| ---- | -------- | ---- |
| 海报标题 `poster_title` | 每条不超过 16 字 | 只保留一个核心利益点或主题 |
| 朋友圈文案 `moments_post` | 每条 80-120 字 | 重点保留时间、截止、价格、卖点、入口 |
| 社群通知 `group_notice` | 120-180 字 | 信息完整即可，不必堆砌情绪词 |
| 短信 `sms` | 不超过 70 字 | 只保留最关键的事实和动作 |

## 内部通知场景

| 渠道 | 建议长度 | 说明 |
| ---- | -------- | ---- |
| 通知标题 `notice_title` | 不超过 20 字 | 主题清楚即可，不求华丽 |
| 企业微信通知 `wechat_group_notice` | 90-160 字 | 够清楚就停，不写成长文 |
| 邮件标题 | 不超过 20 字 | 明确事项和时间类型 |
| 邮件正文 `email_notice` | 120-220 字 | 可分短段，优先完整性 |
| 短信 `sms` | 不超过 70 字 | 保留时间、动作和联系人 |

## 超长时的压缩顺序

1. 先删形容词和重复表达
2. 再压缩次要卖点
3. 把背景说明移到长文本渠道
4. 最后才考虑删减非核心信息

## 不可因压缩而删除的内容
- 时间
- 截止时间
- 价格
- 联系方式
- 报名入口
- 动作要求

## 审校提醒
- 标题长了，先改标题，不要直接改主卖点
- 短信长了，先删修饰，不要先删联系人
- 朋友圈文案长了，优先保留报名和截止信息
- 内部通知长了，也不能删返岗时间和联系人
